Overview

We'll go through this in class, but the basic steps are simple enough to describe:

  • Make sure you have opened port 30025 on your instance and have an elastic IP set up.
  • Use SSH to attach to your instance.
  • Install nodejs
  • Install git
  • Use git to pull down your project. Run your project.
  • Browse to your elastic IP:30025

If you do not yet have an account on AWS, be sure to use the free tier when you create your account:

  • http://aws.amazon.com/free/

If you are at all confused about how to sign up for the free tier, wait until this subject is discussed in class.

The directory with my scripts for configuring your EC2 instance:

  • Node Install Directory
  • Install Node
  • Install NPM Packages
  • Get most recent npm

The get most recent npm script has failed on some systems, but it worked for me on my ECM instance. And your should look a lot like mine.

Unprotected Private Key

If you get this warning:

$ ssh-add Prog272_AtSchool.pem
@@@@@@@@@@@@@@@@@@@@@@@@@@@@@@@@@@@@@@@@@@@@@@@@@@@@@@@@@@@
@         WARNING: UNPROTECTED PRIVATE KEY FILE!          @
@@@@@@@@@@@@@@@@@@@@@@@@@@@@@@@@@@@@@@@@@@@@@@@@@@@@@@@@@@@
Permissions 0640 for 'Prog272_AtSchool.pem' are too open.
It is required that your private key files are NOT accessible by others.
This private key will be ignored.

Then do this for the Private key that caused the trouble:

chmod 400 Prog272_AtSchool.pem

The .bashrc File

I put this at the bottom of my .bashrc on EC2:

export NODE_PATH=:/home/ubuntu/npm/lib/node_modules

export PATH="$PATH:$HOME/npm/bin"

if [ -z "$SSH_AUTH_SOCK" ] ; then
    eval `ssh-agent`
fi

# Uncomment the next line if you want to load your private key automatically
# every time you log into your bash shell. That is, every time you sign on
# to your ubuntu server

# ssh-add $HOME/.ssh/id_rsa
