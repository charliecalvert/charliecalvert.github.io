Download and Load Local Images

So far we have been linking to images stored on remote sites such as the Wikimedia Commons. Some sites, however, ask you to download the pictures rather than having you link to their site directly. This saves the site money, as they don't have to pay for hosting your image. There will, of course, also be times when you want to use images that you took yourself. This assignment describes how to work with images that you have downloaded to your repository.

Download an Image

You have to take an extra step or two when using pixabay images. First, download the image, then link to them locally.

Go to a page like this:

https://pixabay.com/en/utah-mountain-sky-nature-440520/

Download the image into a subdirectory of your repository called images.

Load the Image

Create a file called download-image.html in the root of your repository. Prep it as you usually would, and link in a CSS file of your choice, but it must be a file that you created by hand.

In your new HTML document, load the image you downloaded like this: 

            <img src="images/utah-440520_640.jpg" alt="foo">

Note that this particular image does not require attribution.

Relative Paths

One of the most important subjects in web development involves composing relative paths. This is an absolute path:

<http://www.elvenware.com/foo.html>

Paths like this can be linked from anywhere in the web. But suppose you are trying to link to a document on your own web site. In those casees, you don't need to link in the name of your website because you are already on your website. Instead, you can just enter the name of the file you want to link to. The trouble comes when you are in a subdirectory and want to link to a document that is closer to the root of your site, or that is in a directory that closer to the root.

From your theme directory, for instances, the load statement to the a file in the images directory would involve a relative path: 

            <img src="../images/utah-440520_640.jpg" alt="utah mountains">
