Overview

We need a way for the user to set up a menu, table of contents or index to support navigation through the:

  • Site
  • Document
  • Across the web to remote sites

In this assignment we will set up navigation for the document and the footer. We will tackle setting up the navigation for the entire site and remote sites in a separate assignment.

The nav Tag

Just as we put headers in a header element and footers in a footer element, we should put navigation elements in the HTML5 block level nav element. It's supported by all modern browsers and IE 9 and higher.

Inside the nav element we put links to locations in our documents. Typically, that would be an unordered list or simply a set of anchor tags.

                
<nav>
    <ul>
        <li><a href="index.html">Home</a></li>
        <li><a href="about.html">About</a></li>
    </ul>
</nav>

We often use the same HTML for various navigation elements regardless of the role they play. For instance, a navigation-bar on the top of a site, and a side-bar on the right or left of a site, are created with similar HTML. They could, for instance, both be variants of the unordered list shown above. We distinguish navigation-bars from side-bars by how we style them in our CSS.

I should perhaps add that we frequently use top level navigation-bars for navigation through a site, and use side-bars for navigation through a page. But there are no hard and fast rules here.

Style it

The problem we face now is that the navigation control we have created doesn't look right:

  • Home
  • About

We want it to look like this, with no discs and with the list items arranged horizontally rather than vertically:

  • Home
  • About

To do that, we style the nav tag and its unordered list:

                
    nav {        
        background-color: aliceblue;
        border: 1px green solid;
        padding: 8px;
        margin: -1px 0px 0px 0px;
    }
    
    nav ul {
        list-style-type: none;
        margin: 0;
        padding: 0;
        
    }
    
    nav li {
        display: inline;
        padding: 8px;    
    }

The key declarations here are:

  • list-style-type: none; on the unordered list.
  • display: inline; on the list items

The first declaration gets rid of the discs. The second sets the display of the list to inline. This means that it is no longer a block display. As a result, each list item is not shown on its on line, it is not shown in its own vertical block. Instead, the list items are spread out horizontally.

Notice also the -1px value in the nav margin declaration. This is done to "collapse" the border between the header or footer and their adjacent nav sections. In a real site, we probably would not include these borders, but I have put them in this document to help you see how the header and footer fit with their nav elements. Just look at the source for this page if you have questions about exactly how this works.

Internal Links

When navigating inside a document, we use internal links. This means we link from one part of a document to another part. Typically we are linking from a TOC to a section of the document.

We have been breaking out document up into three sections:

  • header
  • main
  • footer

Inside the main element we might have multiple sections. We can put ids on each section:

                
    <main>
        <section id="jazz"> ... </section>
        <section id="rock"> ... </section>
        <section id="classical"> ... </section>
    </main>

Now we can link to each section in our anchor tags like this:

                
    <nav>
        <ul>
            <li><a href="#jazz">Jazz</a></li>
            <li><a href="#rock">Rock</a></li>
            <li><a href="#classical">Classical</a></li>
        </ul>
    </nav>

Now, when the user clicks on a link, they should be taken to appropriate part of the document.
