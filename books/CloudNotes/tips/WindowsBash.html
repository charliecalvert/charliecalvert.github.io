Overview

This is bash for Windows in the Windows Subsystem for Linux (WSL).

The key is to create a large number of symbolic links. Just put all your code in some regular Windows directory. If it is already there, then leave it there. Create symbolic links to the directories where you store your code.

I've been able to edit my code using Windows tools and compile it using WSL. The key is not be actively uising it Linux while you are editing it. For instance, don't use nodemon, since it keeps your files open, or at least keeps some kind of link to them. Instead, use something like node bin/www to launch your projects.
